# 页面构图与角色定义

这些不是固定模板，而是写提示词时的构图角色。先选页面角色，再写提示词。

---

## 页面角色定义

| 角色 | 用途 | 构图 | 图片内文字量 |
|------|------|------|------------|
| `cover` | 封面，一眼说明主题和气质 | 大标题 + 视觉锚点 + 少量元信息 | 标题 1 行 |
| `method-overview` | 方法总览 | 输入 → 核心模块 → 输出 | 5-9 个短标签 |
| `mechanism-detail` | 单个核心机制拆解 | 局部放大、编号步骤、短标签 | 6-12 个短标签 |
| `comparison` | 新旧方法/方案对比 | 左右对照、差异高亮 | 4-8 个短标签 |
| `evidence` | 实验或数据证据 | 大数字 + 小图表 + 解释标签 | 1 个大数字 + 2-4 个短解释 |
| `architecture` | 系统/模块架构图 | 分层/分组、依赖箭头 | 4-8 个模块标签 |
| `data-flow` | 数据流/管道图 | 数据源→处理→存储→消费 | 4-8 个节点标签 |
| `takeaway` | 结论和要点 | 1 句结论 + 3 个要点 | 1 主句 + 3 短点 |

---

## 推荐节奏

### 4-6 页推荐结构
```
cover → method-overview → mechanism-detail → comparison/evidence → takeaway
```

### 6-10 页推荐结构
```
cover → method-overview → mechanism-detail (×2-3) → evidence → comparison → takeaway
```

---

## 构图规则

### Cover（封面）
- 文字少，主视觉强
- 标题居中或偏上
- 视觉锚点（与论文核心概念相关的抽象/具象图）

### Method Overview（方法总览）
- 中间主图必须占 65%-80% 画面
- 输入在左/上，输出在右/下
- 核心模块在中间，连接线清晰
- 关键创新点用强调色/粗线标记

### Mechanism Detail（机制细节）
- 每页只拆一个机制
- 局部放大关键区域
- 逐步展开（编号 ① ② ③）
- 补充维度信息、公式短注

### Comparison（对比）
- 左右结构必须对称
- 差异点要显眼（不同颜色/虚线框标记差异区域）
- 上方标题，下方结论

### Evidence（证据/数据）
- 数字要大（视觉重点）
- 解释要短（不写长段文字）
- 可以用简化的小图表辅助

### Takeaway（结论）
- 不要再引入新概念
- 回到封面提出的问题形成闭环

---

## 留白控制

- 封面和章节分隔页可以有较多留白（30%-40%），营造呼吸感
- 方法总览页主图应占 65%-80%
- 机制细节页主图应占 75%-85%
- 不允许大面积无意义空白（看起来像没做完）
- 留白服务于视觉层级，不是偷懒

---

## 文字密度建议

| 页面类型 | 文字量 | 具体限制 |
|---------|--------|---------|
| cover | 极少 | 标题 1 行 ≤16 汉字，副标题 1 行 |
| method-overview | 中 | 5-9 个短标签，每个 ≤8 汉字 |
| mechanism-detail | 中高 | 6-12 个短标签，每个 ≤8 汉字 |
| comparison | 中 | 左右各 3-5 个标签 |
| evidence | 少 | 1 个大数字 + 2-4 个短解释 |
| architecture | 中 | 4-8 个模块标签 |
| takeaway | 少 | 1 句结论 + 3 个短点 |

如果文字超过限制，优先拆页，不要强行让生图模型写长段落。
