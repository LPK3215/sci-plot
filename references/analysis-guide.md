# 项目/论文/代码分析指南

## 分析目标

深度理解用户提供的任何与研究相关的内容（论文、项目、算法、代码、文档），提取出"可以画成图的核心信息"。

---

## 论文分析清单

### 第一步：快速概览
- 论文标题、作者、发表时间/会议
- 一句话核心贡献
- 解决什么问题

### 第二步：方法拆解
- 方法的整体输入→处理→输出流程
- 拆出每一个核心模块/组件
- 每个模块的内部机制
- 与已有方法的关键差异

### 第三步：实验分析
- 最重要的实验结果（不是所有结果）
- 与 baseline 的对比
- 最能说明方法有效的那个数字/图表

### 第四步：识别"可图解"
- 方法的整体架构 → 总览图（1 页）
- 每个核心创新机制 → 机制拆解图（每个 1 页）
- 关键实验结果 → 数据图/对比图（1-2 页）
- 不重要或文字就能说清楚的内容 → 跳过

---

## 项目代码分析清单

### 第一步：项目结构
- 项目用途/目标
- 核心目录结构
- 主要入口文件

### 第二步：核心模块
- 关键模块/类的职责
- 模块之间的依赖关系
- 数据流方向

### 第三步：算法逻辑
- 核心算法的输入/输出
- 主要步骤和决策点
- 关键数据结构

### 第四步：识别"可图解"
- 系统架构 → 架构图（1 页）
- 核心算法流程 → 流程图（1-2 页）
- 数据流/管道 → 数据流图（1 页）
- 模块关系 → 组件关系图（1 页）

---

## 算法分析清单

### 第一步：算法要素
- 输入格式和约束
- 输出格式和含义
- 核心思想（一句话）

### 第二步：算法步骤
- 逐步拆解算法流程
- 每步的关键操作
- 时间复杂度/空间复杂度

### 第三步：与 baseline 对比
- 比什么方法好在哪
- 关键创新点

### 第四步：识别"可图解"
- 算法总览流程 → 流程图（1 页）
- 核心创新步骤 → 细节图（1-2 页）
- 与 baseline 对比 → 对比图（1 页）
- 复杂度/性能 → 数据图（1 页）

---

## 文档分析清单

### 第一步：文档定位
- 文档类型（技术文档/API 文档/设计文档/白皮书）
- 核心主题

### 第二步：结构化信息
- 架构/设计概览
- API/接口清单
- 数据模型
- 关键流程

### 第三步：识别"可图解"
- 系统架构 → 架构图
- API 调用流程 → 流程图
- 数据模型关系 → ER 图/关系图
- 业务流程 → 泳道图/流程图

---

## 分析自检清单

分析完成后逐条确认：

- [ ] 能否用一句话说清楚这东西是做什么的？
- [ ] 核心创新/亮点是什么（1-3 个）？
- [ ] 整体流程/架构线是否清晰（输入→处理→输出）？
- [ ] 每个关键模块内部是否理解？
- [ ] 最有说服力的结果/数据是什么？
- [ ] 哪些内容确实需要画图，哪些文字即可？
