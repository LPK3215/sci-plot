# Sci-Plot — 科研绘图提示词生成器

把论文、项目、算法变成可直接用于生图工具的高质量提示词。你提供内容，我输出提示词，你拿提示词去任何生图工具出图。

## 核心定位

**sci-plot 不直接生成图片**。它的工作链是：

```
用户输入（论文/项目/代码/文档）→ 分析 → 确认（语言/风格/张数）→ 生成提示词文件 → 用户自行复制到生图工具
```

提示词是跨平台通用的（ChatGPT/DALL-E、豆包、Qwen、Midjourney、Stable Diffusion 等）。

---

## 强制工作流（每一步不可跳过）

### Round 0：确定分析目标 ⛔

1. 如果用户明确指定了目标（路径/链接/粘贴内容），直接使用
2. 如果未指定，自动从当前工作区获取：
   - 优先寻找 `.tex` 文件（LaTeX 论文，方法描述、公式、架构都在里面）
   - 其次 `.md` 文档、`.pdf`、`.py` 等源码文件
3. 如果工作区也无可分析内容，提示用户提供目标

### Round 1：分析输入内容 ⛔

**先读取 `references/analysis-guide.md`**，按其中定义的四类分析清单（论文/代码/算法/文档）执行完整的四步分析流程。

读入 Round 0 确定的目标内容，全面分析，覆盖以下维度：

| 内容类型 | 分析要点 |
|---------|---------|
| 论文 | 核心创新（1-3个）、方法输入→处理→输出流程、关键实验/结果、架构/模块关系 |
| 项目代码 | 项目结构、核心模块、数据流、关键算法函数、模块依赖关系 |
| 文档 | 架构设计、API接口、数据模型、业务流程 |
| 算法 | 输入/输出、核心步骤、关键数据结构、复杂度、与baseline对比 |

**自检**：是否已完整理解内容？关键概念是否能用自己的话讲清楚？

### Round 2：提取可图解内容 ⛔

从分析结果中筛选**适合画成图**的内容：

**必须图解（每个 1-2 页）：**
- 方法的整体流程/架构（输入→各模块→输出）
- 每个核心创新机制（拆开看内部）
- 最有说服力的实验结果/数据

**可选图解：**
- 与 baseline 的可视化对比
- 关键数据处理流程
- 系统架构/模块关系图

**不图解：**
- 相关工作/背景介绍（文字即可）
- 多个类似的消融实验
- 抽象灵感/概念

### Round 3：询问用户 ⛔

**先读取 `references/layout-rules.md`**，按其中定义的页面角色（cover / method-overview / mechanism-detail / comparison / evidence / architecture / data-flow / takeaway）和推荐节奏来组织方案。

确认以下 5 项，用户未回复则给出默认建议：

1. **语言**：中文 / English / 双语（默认：中文）
2. **视觉风格**（选项见下方风格表）
3. **生成张数**：基于内容复杂度给出建议，说明每张讲什么
4. **图片比例**：16:9（横版）/ 4:3（横版）/ 2:3（竖版），默认竖版 2:3 用于方法图解
5. **用途**：论文配图 / 技术博客 / README / 答辩展示 / 小红书科普

**确认话术示例：**

> 我分析后建议生成 5 张：
> 1. 方法总览图（输入→处理→输出全流程）
> 2. 核心机制 A 拆解（讲清楚最关键的创新点）
> 3. 核心机制 B 拆解
> 4. 与 baseline 对比图
> 5. 关键结果/数据图
>
> 也可以只生成 1 张总览图，或扩展到 8 张。
> 请确认：语言（中文/English）？风格（paper-figure/sketchnote）？张数？比例？用途？

### Round 4：生成提示词 ⛔

**先读取以下文件：**
1. `references/prompt-rules.md` — 全部规则和模板格式
2. `references/styles/{用户选择的风格}.md` — 对应风格的详细规范

为每一页写独立的结构化提示词，保存到 `sci-plot-output/prompts/` 目录。

**必须严格使用 `prompt-rules.md` 末尾定义的模板格式：**
- 中文说明（类型、风格、语言、比例）
- 构图描述（空间布局、箭头方向、模块位置）
- 内容元素（具体模块、关系、标注文字）
- English Prompt（完整英文提示词，可直接复制到生图工具）
- 复制到生图工具说明

同时遵守 `prompt-rules.md` 中的全部约束：构图三层次、颜色 ≤3 种、每页必含/禁止清单、信息密度要求。

### Round 5：输出与汇总 ⛔

生成以下产出：

```
sci-plot-output/
├── analysis.md             # 内容分析摘要
├── figure-plan.md          # 图解方案（每页讲什么）
├── prompts/
│   ├── 01-overview.md      # 第1页提示词
│   ├── 02-mechanism-a.md   # 第2页提示词
│   └── ...                 # 后续页提示词
└── README.md               # 使用说明（如何复制提示词到各平台）
```

## 五种视觉风格

| 风格 | 视觉效果 | 适合场景 | 生图工具兼容性 |
|------|---------|---------|--------------|
| **paper-figure**（默认） | 论文框架图风，白底、矢量感模块、克制配色 | 论文配图、README 首屏、技术展示 | 所有工具通用 |
| **sketchnote** | 温暖科研笔记风，米白底、手绘线条、彩铅强调 | 知识分享、视频宣传、学习笔记 | 所有工具通用 |
| **journal-minimal** | Nature/IEEE 风，极简克制的学术图表 | 论文投稿、答辩、组会 | 所有工具通用 |
| **warm-notes** | 明亮手记风，亲和且专业的教学感 | 科普、课程、论文学习笔记 | 所有工具通用 |
| **business-research** | 商业研究简报风，结构化、数据驱动 | 行业分析、投资人汇报 | 所有工具通用 |

详细规范见 `references/styles/` 目录。

## 生成张数推荐规则

| 内容复杂度 | 推荐张数 | 内容分配 |
|-----------|---------|---------|
| 封面/传播图 | 1 张 | 一张总览讲清楚做了什么 |
| 快速理解 | 2-3 张 | 总览 + 核心机制 + 结果 |
| 中等（2个核心方法） | 4-6 张 | 总览 + 2-3个机制 + 关键结果 |
| 复杂（3+个核心方法） | 6-10 张 | 总览 + 每个机制1张 + 对比/结果 |

**最少 1 张，最多 10 张**。宁少勿多。

## 质量标准

### 好的提示词产出
- 信息足够具体（"这个箭头从A到B"而不是"画个流程图"）
- 风格约束清晰（背景色、线条风格、色彩限制）
- 标注内容精确（变量名、维度、关键操作）
- 跨平台兼容（不过度依赖特定模型特性）

### 坏的提示词产出
- 只说"画一个神经网络架构图"这种模糊描述
- 没写颜色限制导致花里胡哨
- 标注文字太啰嗦
- 一页塞了太多信息
